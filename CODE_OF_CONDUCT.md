# Code of Conduct

## Our Standard

ELSPETH is built for high-assurance work where correctness, evidence, and clear
communication matter. Participants are expected to:

- be respectful and professional;
- assume good faith while still asking for evidence;
- keep criticism specific to code, documentation, behaviour, or decisions;
- avoid harassment, intimidation, discrimination, and personal attacks;
- treat security reports, private data, and audit evidence with care.

## Unacceptable Behaviour

Unacceptable behaviour includes:

- harassment, abuse, threats, or discriminatory language;
- publishing another person's private information without permission;
- sharing secrets, exploit details, personal data, or sensitive audit material
  in public channels;
- sustained disruption of project discussions;
- misrepresenting the assurance, compliance, or production status of the
  project.

## Enforcement

Maintainers may remove, edit, or reject comments, commits, issues, pull
requests, and other contributions that violate this code of conduct.
Maintainers may also temporarily or permanently restrict participation.

Security-sensitive conduct issues should follow `SECURITY.md` if public
disclosure would create risk.

## Attribution

This code of conduct is intentionally short and project-specific. It is
compatible in spirit with common open-source community standards while keeping
the release-facing public-sector context explicit.
