# AWS S3 Sink Plugin Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a production-safe `aws_s3` sink for CSV, JSON arrays, and JSONL with bounded spooled serialization, AWS-default credentials, race-safe conditional writes, audit-primary redacted failures, and real-S3 acceptance.

**Architecture:** `src/elspeth/plugins/sinks/aws_s3_sink.py` first lands as an unregistered config/serialization module with no `BaseSink` subclass. The complete `AWSS3Sink` then registers atomically with its S3 call, audit, lifecycle, catalog, and test surfaces. Candidate objects serialize incrementally into an 8 MiB `SpooledTemporaryFile`; first `overwrite=False` writes use `IfNoneMatch="*"`, and later cumulative rewrites use the last confirmed ETag through `IfMatch`, so another writer is never silently clobbered.

**Tech Stack:** Python 3.13, Pydantic v2, boto3/botocore, lazily imported Jinja2, tempfile spooling, stdlib CSV/JSON, existing sink/audit infrastructure, pytest/Hypothesis.

**Executable now:** No. Filigree `elspeth-74717426b7` remains blocked by Plan 06 `elspeth-7fe6aa531f` and the Plan-08 core gate `elspeth-c0103e6c88`. Review approval does not authorize bypassing those edges. For a strictly linear implementation sequence use `08 gate baseline -> 08A`, then `02 + 08A -> 06 -> 07`, not document-number order.

**Hard dependencies and ordering:**

- Plan 06 supplies `build_s3_client(region_name, endpoint_url)`, the initial `aws` extra, `AWSS3*` naming, retry/timeouts, and the shared public-error/close hygiene contract.
- Plan 08A Tasks 1-3 must already be committed. Tasks 1-2 are the load-bearing validation gate and Task 3 completes immediate composer-mutation defenses. **Global constraints:**

- Public names are `AWSS3Sink`, `AWSS3SinkConfig`, and `CSVWriteOptions`; the plugin name is `aws_s3` and determinism is `IO_WRITE`.
- Pipeline config has only `bucket`, `key`, `format`, `overwrite`, CSV/header/schema options, `region_name`, `endpoint_url`, `max_object_bytes`, and `max_record_chars`. It has no credential, provider, nested client-config, or arbitrary boto kwargs surface.
- Every top-level registered config field uses `Field` with a concrete non-empty description. `AWSS3SinkConfig` declares `_plugin_component_type: ClassVar[str | None] = "sink"`; `DataPluginConfig` supplies strict `extra="forbid"` and required schema behavior.
- Bucket, region, endpoint, key-template, and rendered-key validation matches Plan 06. The rendered key is revalidated after Jinja expansion: nonblank, no NUL/control character, and at most 1024 UTF-8 bytes. Web-authored non-null `endpoint_url` is rejected by Plan 08; omission and explicit `null` remain allowed.
- `max_object_bytes` defaults to 256 MiB, is positive, and has a 1 GiB ceiling. `max_record_chars` defaults to 1,000,000 decoded characters and has an 8,000,000-character ceiling, bounding the temporary line/scalar allocation a CSV/JSON encoder may need. Serialization writes through `SpooledTemporaryFile(max_size=8 * 1024 * 1024, mode="w+b")`; no whole-object `bytes`, list-of-chunks, or unbounded JSON document string is retained.
- A non-serializable, non-finite, unencodable, or over-record-limit incoming row is a Tier-2 data fault and is diverted with a static reason according to `on_write_failure`. The cumulative object exceeding `max_object_bytes` is a sink-capacity failure for the whole candidate write; it does not silently drop whichever row happened to cross the boundary.
- `overwrite=True` uses unconditional `PutObject`. `overwrite=False` uses `IfNoneMatch="*"` before this sink has a confirmed remote ETag and `IfMatch=<last confirmed ETag>` for every later cumulative rewrite. Missing/malformed success ETags fail closed. HTTP 412/409 or a stale ETag never falls back to an unconditional write.
- A generic transport/provider failure can have an ambiguous remote outcome. The sink records a safe failure, marks itself poisoned, raises static `S3WriteOutcomeUnknownError`, and refuses subsequent writes; it does not claim retry-idempotence.
- Provider messages, response bodies, endpoint URLs, object contents, credentials, and raw causes never enter public exceptions, audit, telemetry, or logs. Safe exceptions are raised after leaving raw provider/recorder handlers, with `__cause__` and `__context__` absent. Controlled size failures expose bounded numeric observed/limit facts only.
- Both success- and failure-path `ctx.record_call` writes are audit-primary. Existing `TIER_1_ERRORS` propagate; any other recorder failure poisons the sink and becomes a redacted `AuditIntegrityError` outside the recorder handler. Audit failure after a confirmed PUT never commits the candidate row buffer and never permits another write.
- `close()` first marks closed, detaches the client, clears row/key/ETag/poison state, then closes the detached client exactly once. A close failure becomes a static sanitized error after cleanup; repeated `close()` is a no-op.
- No normal logging is added. Landscape call records are the probative source of truth; Wardline and explicit boundary tests remain mandatory.

`CSVWriteOptions` uses `extra="forbid"` and descriptions for `delimiter`, `encoding`, and `include_header`. The config rejects blank/operator-placeholder bucket and key-template values; bounds bucket text to 2048 characters and template source to 4096 UTF-8 bytes; rejects template NUL/control characters; validates region as nonblank, at most 64 characters, and `[A-Za-z0-9-]+`; and validates endpoint exactly like Plan 06 (original length at most 2048, HTTP(S), hostname required, no userinfo/query/fragment/whitespace/control). `_load_jinja()` lazily imports `StrictUndefined`, `TemplateSyntaxError`, and `SandboxedEnvironment` and gives an actionable `elspeth[aws]` error when absent. Validation compiles the template; runtime revalidates the final key after `run_id`/UTC `timestamp` expansion.

Serialization interface:

```python
@dataclass(slots=True)
class _SerializedObject:
    body: BinaryIO
    size_bytes: int
    content_hash: str
    checksum_sha256_b64: str
    # idempotent close() and context-manager ownership

def _serialize_rows_to_spool(
    rows: Iterable[Mapping[str, Any]],
    *,
    format: Literal["csv", "json", "jsonl"],
    csv_options: CSVWriteOptions,
    fieldnames: Sequence[str],
    max_object_bytes: int,
    max_record_chars: int,
) -> _SerializedObject: ...
```

The helper writes at most 64 KiB per spool call and updates SHA-256/byte count before accepting a chunk. Every format first walks one row with a conservative character budget and rejects before the encoder can materialize more than `max_record_chars`; JSON/JSONL then use `JSONEncoder(allow_nan=False).iterencode()`, while CSV writes the bounded header/rows through the configured encoder. On `total > max_object_bytes`, close the spool and raise `S3ObjectSizeLimitError(observed_bytes, limit_bytes)`; on success rewind and transfer ownership. Serialization/diversion errors never include row values.

Lifecycle state is `_s3_client`, `_buffered_rows`, `_resolved_key`, `_remote_etag`, `_poisoned`, and `_closed`. `write()` rejects closed/poisoned reuse; resolves contract/display headers; preflights each incoming row and diverts JSON/CSV encoding, non-finite, non-serializable, and over-record-limit faults with static reasons; freezes/revalidates the key; serializes cumulative accepted rows without duplicating the prior buffer; and owns the spool through the call. The serialization helper repeats the record cap as a defensive invariant, while cumulative `max_object_bytes` failure aborts the whole write and leaves both remote and buffered state unchanged.

Upload contract:

1. Rewind the spool and send `Bucket`, `Key`, seekable `Body`, exact `ContentLength`, and `ChecksumSHA256`.
2. `overwrite=True` adds no condition. First `overwrite=False` adds only `IfNoneMatch="*"`; later writes add only `IfMatch=<confirmed ETag>`.
3. Use Plan 06's lazy client. Validate the response mapping; under `overwrite=False` require ETag to be exact printable ASCII, 1-1024 UTF-8 bytes, and retain it before auditing.
4. Success audit uses `provider="aws_s3"`, `call_type=HTTP`, request `{operation, bucket, key, overwrite, condition}` where `condition` is only `none`, `if_none_match`, or `if_match`, and response `{size_bytes, content_hash}`. It never records endpoint, body, checksum input, ETag value, or credentials. Failure audit uses the same bounded request shape and a normalized class only.
5. Only after audit succeeds, extend `_buffered_rows` and return an `ArtifactDescriptor`. Always close the spool.

Failure contract:

- Normalize exception class names to `[A-Za-z_][A-Za-z0-9_]{0,127}`. Exact allowlisted `Error.Code` values `PreconditionFailed`/`ConditionalRequestConflict` become static `S3ConditionalWriteRejectedError`; definite request rejections such as `AccessDenied`/`NoSuchBucket`/`InvalidRequest` become static `S3SinkWriteError`.
- Transport, unknown, malformed-response, and other ambiguous outcomes poison the instance and become static `S3WriteOutcomeUnknownError`. No condition failure falls back to unconditional PUT.
- `ImportError` from the shared builder preserves its actionable extra message; `TIER_1_ERRORS` propagate.
- `_record_s3_call` applies audit primacy to success/error. Any non-Tier-1 recorder failure is reduced to normalized class, poisons the sink, and becomes static `AuditIntegrityError` outside its handler, without cause/context.
- A confirmed PUT followed by malformed response or audit failure never commits buffered rows and permanently poisons the instance.
- `close()` marks closed, detaches the client, clears all state, then closes once. A close failure becomes static `S3ClientCloseError` outside its handler; repeated close is a no-op.
