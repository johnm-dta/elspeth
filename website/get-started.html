Reviewable by design

From install to a validated run.

Install the CLI, declare a pipeline, and validate it before it runs. What you read is what runs — and what the audit trail records.

1

Install

Python 3.12+. Clone and install with uv.

$ git clone https://github.com/johnm-dta/elspeth.git && cd elspeth
$ uv venv && source .venv/bin/activate
$ uv pip install -e ".[dev]"
2

Validate before you run

Preflight checks graph structure, route targets, schema compatibility, and secret references.

$ elspeth validate --settings pipeline.yaml
3

Run & explain

Execute the pipeline, then explore why any row reached its destination.

$ elspeth run --settings pipeline.yaml --execute
$ elspeth explain --run <run_id> --row <row_id>
A pipeline is just declared primitives

pipeline.yaml

Every edge is named; every gate is pure config. Prefer not to write YAML? The Web Composer lets you describe the workflow in conversation instead — it builds the same artifact.

# pipeline.yaml
source:
  plugin: csv
  on_success: validated
  options: { path: data/submissions.csv }

transforms:
- name: classify
  plugin: llm
  input: validated
  on_success: classified

gates:
- name: safety_gate
  input: classified
  condition: "row['risk_score'] > 0.8"
  routes: { "true": review, "false": approved }

sinks:
  approved: { plugin: csv }
  review:   { plugin: review_queue }

landscape:
  url: sqlite:///./audit.db
