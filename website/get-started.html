Reviewable by design

From install to a validated run.

Install the CLI, declare a pipeline, and validate it before it runs. What you read is what runs — and what the audit trail records.

Install and run from the CLI

1

Install

Python 3.12+. Clone and install with uv.

$ git clone https://github.com/johnm-dta/elspeth.git && cd elspeth
$ uv venv && source .venv/bin/activate
$ uv pip install -e ".[dev]"
2

Validate before you run

Preflight checks graph structure, route targets, schema compatibility, and secret references.

$ elspeth validate --settings pipeline.yaml
✅ Pipeline configuration valid!
  Sources: submissions=csv
  Transforms: 1
  Aggregations: 0
  Sinks: approved, review
  Graph: 5 nodes, 4 edges
3

Run & explain

Execute the pipeline, then explore why any row reached its destination.

$ elspeth run --settings pipeline.yaml --execute   # → status: completed
$ elspeth explain --run <run_id> --row <row_id>
Outcome:   SUCCESS
Path:      GATE_ROUTED
Sink:      approved
A pipeline is just declared primitives

pipeline.yaml

Every edge is named; every gate is pure config. Prefer not to write YAML? The Web Composer lets you describe the workflow in conversation instead — it builds the same artifact.

# pipeline.yaml — passes: elspeth validate --settings pipeline.yaml
sources:
  submissions:
    plugin: csv
    on_success: validated
    options:
      path: data/submissions.csv
      schema: { mode: observed }   # infer types from the data
      on_validation_failure: discard      # no silent defaults — declare it

transforms:
- name: classify
  plugin: llm
  input: validated
  on_success: classified
  on_error: discard
  options:
    provider: openrouter
    model: anthropic/claude-sonnet-4.6
    api_key: ${OPENROUTER_API_KEY}   # from the OPENROUTER_API_KEY env var
    prompt_template: "Score risk 0-1: {{ row }}"
    schema: { mode: observed }

gates:
- name: safety_gate
  input: classified
  condition: "row['risk_score'] > 0.8"
  routes: { "true": review, "false": approved }

sinks:
  # "review" is a destination role, not a plugin type — route it to a real sink
  approved: { plugin: csv, on_write_failure: discard, options: { path: data/approved.csv, schema: { mode: observed } } }
  review:   { plugin: csv, on_write_failure: discard, options: { path: data/needs_review.csv, schema: { mode: observed } } }

landscape:
  url: sqlite:///./audit.db
Prefer not to write YAML?

Launch the Web Composer.

The Web Composer ships with ELSPETH. Start the local web app, then build a pipeline in conversation — it targets the same engine, contracts, and Landscape audit trail as the CLI.

Run the local server

Starts a FastAPI app bound to your machine. Open the printed address in a browser to start composing.

$ elspeth web
# Composer served at http://127.0.0.1:8451
